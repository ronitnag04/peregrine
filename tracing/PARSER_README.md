# Trace Parser README

## Overview

`parse_trace.cpp` is a C++ parsing script that reads CSV trace files generated by the Pin tool (`peregrine-trace.cpp`) and converts them into structured instruction data for further analysis. The parser provides a convenient way to access instruction-level information including registers, memory accesses, and dependencies.

## Functionality

The parser performs the following operations:

1. **CSV Parsing**: Reads and parses CSV files with proper handling of quoted fields and escaped characters
2. **Data Structure Creation**: Converts CSV rows into `instruction_data_t` structs containing:
   - Instruction pointer (IP)
   - Assembly code
   - Instruction category and opcode
   - Branch type information
   - Synchronization point flags
   - Register dependencies (read/write)
   - Memory access information (read/write addresses with sizes)
   - Register and memory dependency chains (dependent instruction IPs)

3. **Selective Printing**: Allows you to print specific ranges of instructions for analysis
4. **Structured Output**: Displays all instruction details in a human-readable format

## Data Structures

### `mem_access_t`
```cpp
typedef struct {
    unsigned long addr;    // Memory address
    unsigned int size;     // Access size in bytes
} mem_access_t;
```

### `instruction_data_t`
```cpp
typedef struct {
    unsigned long ip;                          // Instruction pointer
    std::string assembly;                      // Assembly code
    std::string category;                      // Instruction category (e.g., DATAXFER, CALL)
    std::string opcode;                        // Instruction opcode (e.g., MOV, CALL_NEAR)
    std::string branch_type;                   // Branch type (direct_conditional, direct_unconditional, indirect, or empty)
    bool inst_sync;                            // True if instruction is a synchronization point
    std::vector<std::string> read_registers;    // Registers read by the instruction
    std::vector<std::string> write_registers;   // Registers written by the instruction
    std::vector<unsigned long> reg_dependent_ips;  // IPs of instructions this depends on (via registers)
    std::vector<mem_access_t> read_addresses;   // Memory addresses read
    std::vector<mem_access_t> write_addresses; // Memory addresses written
    std::vector<unsigned long> mem_dependent_ips;   // IPs of instructions this depends on (via memory)
} instruction_data_t;
```

## Usage

### Basic Syntax
```bash
./parse_trace [CSV_FILE] [OPTIONS]
```

### Command-Line Options

| Option | Description | Example |
|--------|-------------|---------|
| `--range START END` | Print instructions from START to END (inclusive) | `--range 0 10` |
| `--start START --num NUM` | Print NUM instructions starting from START | `--start 100 --num 5` |
| `--help` or `-h` | Display usage information | `--help` |

### Examples

#### Print first 5 instructions (default behavior)
```bash
./parse_trace trace.csv
```

#### Print instructions 0 to 10 (inclusive)
```bash
./parse_trace trace.csv --range 0 10
```

#### Print 5 instructions starting from index 100
```bash
./parse_trace trace.csv --start 100 --num 5
```

#### Print instructions 50 to 75
```bash
./parse_trace trace.csv --range 50 75
```

#### Use a different CSV file
```bash
./parse_trace my_trace.csv --range 0 5
```

## CSV File Format

The parser expects CSV files with the following columns (in order):

1. **IP**: Instruction pointer (hex format: `0x...`)
2. **Assembly**: Assembly code (may be quoted)
3. **Category**: Instruction category
4. **Opcode**: Instruction opcode
5. **Branch Type**: Branch type or empty
6. **Instruction Sync**: `true` or `false`
7. **Read Registers**: Semicolon-separated register names (e.g., `rip;rsp`)
8. **Write Registers**: Semicolon-separated register names
9. **Register Dependent IPs**: Semicolon-separated hex IPs (e.g., `0x72b5241aa293`)
10. **Read Addresses**: Semicolon-separated addresses in format `0xADDR(SIZE)` (e.g., `0x7ffe8bb8af68(8)`)
11. **Write Addresses**: Semicolon-separated addresses in format `0xADDR(SIZE)`
12. **Memory Dependent IPs**: Semicolon-separated hex IPs

### 6. Accessing the Instruction Vector

The `instructions` vector (of type `std::vector<instruction_data_t>`) is available after parsing and contains all instruction data. You can:

- Iterate through all instructions
- Access specific instructions by index: `instructions[0]`
- Use STL algorithms: `std::count_if`, `std::find_if`, etc.
- Build dependency graphs using `reg_dependent_ips` and `mem_dependent_ips`

### Example: Building a Dependency Graph

```cpp
#include <map>
#include <set>

void build_dependency_graph(const std::vector<instruction_data_t>& instructions) {
    std::map<unsigned long, std::set<unsigned long>> graph;
    
    for (const auto& inst : instructions) {
        std::set<unsigned long> deps;
        
        // Add register dependencies
        for (auto dep_ip : inst.reg_dependent_ips) {
            deps.insert(dep_ip);
        }
        
        // Add memory dependencies
        for (auto dep_ip : inst.mem_dependent_ips) {
            deps.insert(dep_ip);
        }
        
        if (!deps.empty()) {
            graph[inst.ip] = deps;
        }
    }
    
    // Use graph for analysis
    std::cout << "Dependency graph has " << graph.size() << " nodes" << std::endl;
}
```

## Troubleshooting

### Common Issues

1. **"Could not open file"**: Ensure the CSV file path is correct and the file exists
2. **Segmentation fault**: Check that instruction indices are within bounds when using `--range` or `--start`
3. **Parse errors**: Verify the CSV file format matches the expected structure
4. **Compilation errors**: Ensure you're using a C++11 compatible compiler

### Debugging Tips

- Add debug output in `parse_csv_line()` to see what's being parsed
- Check the CSV header line matches expected format
- Verify semicolon-separated lists are being split correctly
- Use `--range 0 0` to test parsing of a single instruction

## Integration with Other Tools

The parser can be easily integrated with other analysis tools:

1. **Pipe output**: `./parse_trace trace.csv --range 0 10 | grep "Branch"`
2. **Redirect to file**: `./parse_trace trace.csv --range 0 100 > output.txt`
3. **Use as library**: Extract the parsing functions and include in other projects

## Performance Considerations

- The parser loads all instructions into memory, so very large trace files may consume significant RAM
- For large files, consider adding streaming/iterative parsing
- Current implementation is single-threaded; parallelization can be added for analysis
